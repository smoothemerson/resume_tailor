| main() → generate_tailored_resume() | Job description string passed to LLM client module |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | stdin input | accept | Job description is user-supplied text for their own resume; no third-party input path; QUAL-04 (XML delimiter wrapping) already implemented in llm_client.py Phase 2 |
| T-03-02 | Information Disclosure | success message (stdout) | accept | Prints absolute path of output file; single-user personal CLI tool, no multi-user context, no secrets in path |
| T-03-03 | Denial of Service | empty JD guard | mitigate | Validate job_description.strip() before calling LLM; print error to stderr and exit 1 — prevents sending empty payload to Ollama |
| T-03-04 | Elevation of Privilege | argparse flags | accept | No new flags added per D-07; no flag injection surface; argparse used for --help only |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed in this phase; stdlib only + existing project deps; no slopcheck needed |
</threat_model>

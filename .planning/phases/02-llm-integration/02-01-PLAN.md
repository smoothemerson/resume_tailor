<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> LLM prompt | Job description (arbitrary text) is embedded in the user message sent to Ollama |
| Ollama response -> LaTeX string | LLM output is untrusted; may contain fences, partial content, or non-LaTeX prose |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Job description in prompt | mitigate | QUAL-04: wrap in `<job_description>...</job_description>` XML delimiters; prevents LaTeX special characters ($, %, &, _, #, ^) from being interpreted as LaTeX by the model's attention mechanism |
| T-02-02 | Information Disclosure | Ollama at localhost | accept | All HTTP calls go to 127.0.0.1:11434 only; no external network calls; no user data leaves the machine; no TLS needed for loopback |
| T-02-03 | Denial of Service | LLM response truncation | mitigate | QUAL-03: done_reason="length" check aborts before writing; prevents silent truncated output that would fail to compile |
| T-02-04 | Tampering | LLM injects non-LaTeX output | mitigate | QUAL-01 + QUAL-02: unconditional fence strip then \\documentclass/\\end{document} boundary validation; invalid output raises ValueError before being returned to caller |
| T-02-05 | Denial of Service | Ollama service down at runtime | mitigate | ERR-03: GET /api/tags health check before any prompt construction; fails fast with RuntimeError instead of 300s timeout wait |
| T-02-06 | Elevation of Privilege | Prompt injection via job description | accept | Tool is personal-use CLI; job description is provided by the user themselves; no third-party job description ingestion pipeline in scope for v1 |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this phase; existing `requests` dependency verified in Phase 1 (PKG-01); stdlib `re` and `json` require no install |
</threat_model>

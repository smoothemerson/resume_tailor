<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pyproject.toml → uv build system | Package metadata consumed by uv at install time; no user-controlled input |
| uv tool install → filesystem | Writes to user's tool directory (~/.local/share/uv/tools/); no elevated privileges |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | pyproject.toml — build-system | accept | [build-system] references hatchling, a well-known PyPA-maintained package (approved in RESEARCH.md audit). No untrusted build backend. |
| T-02-02 | Tampering | pyproject.toml — dependencies | accept | Only requests>=2.32.0 is declared as a runtime dependency. Approved in Package Legitimacy Audit. requests 2.32 patched GHSA-9wx4-h78v-vm56 (urllib3 CVE). |
| T-02-03 | Elevation of Privilege | uv tool install | accept | uv tool install runs as the current user, not root. Tool directory is in user home. No sudo required and none should be used. |
| T-02-SC | Tampering | pip/uv package installs | accept | All packages declared in pyproject.toml are pre-approved in RESEARCH.md Package Legitimacy Audit: requests [OK], hatchling [OK], ruff [OK], mypy [OK]. No [ASSUMED] or [SUS] packages. |
</threat_model>

## v1 Requirements — COMPLETED

### Core Flow

- [x] **CORE-01**: User can run the CLI tool that reads the base LaTeX resume from a configurable path in `config.py` — **Validated Phase 1**: read_resume() with Path(__file__)-anchored config path
- [x] **CORE-02**: User can input a multiline job description via terminal prompt (type END on a new line to submit); EOFError treated as submission — **Validated Phase 3**: while-True input loop with END/EOFError break
- [x] **CORE-03**: User sees a progress message before the LLM call starts (so the tool does not appear frozen during cold model load) — **Validated Phase 3**: print(..., flush=True) before generate_tailored_resume()
- [x] **CORE-04**: Tool calls Ollama `/api/chat` (non-streaming) with `num_ctx: 8192`, `timeout=(10, 300)`, and role-separated system/user messages — **Validated Phase 2**: all params confirmed in llm_client.py
- [x] **CORE-05**: Tool writes the tailored LaTeX output to `resumes/output/tailored_resume_YYYYMMDD_HHMMSS.tex` (directory created if missing) — **Validated Phase 1**: write_resume() with datetime strftime + mkdir(parents=True, exist_ok=True)
- [x] **CORE-06**: Tool prints a success message with the full output file path on completion — **Validated Phase 3**: output_path.resolve() in success path

### Output Quality & Safety

- [x] **QUAL-01**: LLM response is unconditionally stripped of markdown code fences before any further processing — **Validated Phase 2**: _strip_fences() called regardless of fence presence
- [x] **QUAL-02**: Output is validated in memory for `\documentclass` presence and `\end{document}` termination before writing to disk — **Validated Phase 2**: _validate_latex() raises ValueError if either missing
- [x] **QUAL-03**: `done_reason` field in Ollama response is checked; if `"length"`, abort with a clear error message — **Validated Phase 2**: done_reason check is step 1 before fence strip
- [x] **QUAL-04**: Job description is wrapped in `<job_description>...</job_description>` XML delimiters in the prompt — **Validated Phase 2**: _build_messages() wraps user content

### Error Handling

- [x] **ERR-01**: If base resume file is not found, tool prints a human-readable error to stderr and exits with code 1 — **Validated Phase 1**: FileNotFoundError caught inline in resume_reader.py
- [x] **ERR-02**: If Ollama connection fails or times out, tool prints a human-readable error to stderr and exits with code 1 — **Validated Phase 2**: RuntimeError raised; Phase 3 CLI catches and exits 1
- [x] **ERR-03**: Tool performs an Ollama health check at startup (`GET /api/tags`) and fails fast with a clear message if Ollama is not reachable — **Validated Phase 2**: _check_ollama_health() called first in generate_tailored_resume()

### Configuration

- [x] **CONF-01**: `config.py` exposes `BASE_RESUME_PATH`, `OLLAMA_MODEL`, `OLLAMA_BASE_URL`, `OUTPUT_DIR`, and `TIMEOUT` as named constants — **Validated Phase 1**: five public constants at module level
- [x] **CONF-02**: All paths in `config.py` are anchored to `Path(__file__)` so the tool works regardless of invocation directory — **Validated Phase 1**: _HERE/_ROOT anchors used throughout

### Packaging

- [x] **PKG-01**: Project is packaged with `pyproject.toml` + `uv`; installable as a `resume-tailor` shell command via `uv tool install .` — **Validated Phase 1**: human UAT confirmed `resume-tailor --help` works after install
